Nice little problem on bitpatterns
The bit-weight of a number is defined as the number of bits in its bit-pattern that are set to 1. e.g., the bit-weight of 4(00000100) as well as that of 8(00001000) is 1; the bit-weight of 3(00000011) is 2 and that of 7(00000111) is 3.

The problem: Given a positive number n, find the smallest number greater than n that has the same bit-weight as n.

Try and code (in C/C++)the smallest AND easiest solution to this. And try using bitwise operations.

This seems to be a topcoder problem.

Try this: let n be the size of the bit pattern, and k be the number of on bits.

1. convert your number to binary, apply next_permutation on it.
2. if next_permutation worked, simply convert it back to decimal and return the result.
3. else, create a vector with a leading 1, and k - 1 trailing 1s and the remaining locations in between set to 0. Convert to decimal and return the result.

int bitPosition(int n)
{
int count = 0;
while(n>1)
{
n/=2;
count++;
}
return count;
}

int nextInt(int n)
{
int a, b, c, d, e, f, g, val;
a = n ^ (n-1);
b = n | a;
c = b + 1;
d = b ^ c;
e = n & d;
f = a & e;
g = (e >> bitPosition(f));
val = c | g;
return(val);
}

if we can write bitPosition(n) where only 1 bit is set in n in O(1), then nextInt(n) is O(1) time algorithm
